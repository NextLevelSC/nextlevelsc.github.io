AWS Introduction to Cloud Computing

This pathway introduces the fundamentals of cloud computing, as well as the AWS cloud computing services. With cloud being a part of everyday life and companies increasingly using cloud solutions to meet their technology needs, it is important to first understand what AWS cloud is and the core concepts, regardless of your background or experience. This pathway will then outline a series of mental models from the AWS Well-Architected Framework, the five pillars of AWS, which are to be considered when working in the cloud. These include: Operational Excellence, Security, Reliability, Performance Efficiency and Cost Optimization. Following this, you will learn about the six main benefits of cloud computing, which also relate back to the five pillars and the AWS values of security, reliability and scalability. Finally, there is an opportunity to explore additional learning pathways for free with Amazon Web Services, providing more insight into the AWS Well-Architected Framework, including related best practices and design principles.

AWS Developer Fundamentals

In this pathway, the development skills used to design, construct, and test applications will be covered. After an overview of the basics of software development and architecture, the concept of DevOps will be introduced and explored in detail, including defining the DevOps model, why it is important, the benefits derived from integrating DevOps as common practice and how it relates to the change management strategy of blue/green deployment. Following this, the additional best practices of CI/CD and infrastructure as code will be explored. Additionally, this pathway will walk through nine different scripting and programming languages and take a deep dive into their purposes. Databases and their corresponding AWS services are also explored, including relational, non-relational and data structures. The final component of this pathway is a module on serverless architectures. The key concepts outlined in this include a definition of serverless, why serverless should be used, several key benefits of going serverless and lastly, best practices to follow when using serverless architectures.

AWS Application Developer

This pathway is an extension of the AWS Developer Fundamentals, diving deeper into software development specifically for applications. It first introduces web and mobile development, with an emphasis on additional resources to explore including SDKs and sample code, hosting websites and applications on certain AWS services and various best practices. Next, platforms will be introduced, including how operating systems and virtualization are used in the cloud. Following this, the module will explore applications in the context of startups and game development. In keeping up with current industry trends and standards, the final module walks through modern application development. Specifically, webhooks and modernization will be explored. Additionally, this module also includes a brief introduction into machine learning and how it can be used to build intelligent applications on AWS.

AWS Cyber Security

This pathway covers the importance of security when working with data in the cloud by outlining the key concepts that should be taken into consideration. The course begins by stepping through security basics in AWS, including the AWS Global Cloud Infrastructure and an outline of the tools and features that can be used to meet or improve security standards. Following this is information regarding identity and access management tips, tutorials and best practices, as well as federation. The pathway then moves to networking with AWS and exploring how you can secure your infrastructure, specifically with the AWS services Virtual Private Cloud (VPC), Direct Connect, Elastic Load Balancing and Route 53. Data encryption is also a crucial aspect of security within AWS. How this can be achieved is explained through AWS Key Management Service. The management of security threats with several AWS services is also explored in this pathway. Finally, the pathway covers cloud compliance and assurance measures, with an additional introduction to AWS security hub.

AWS Startups

In this pathway, the core aspects of getting your startup up and running are covered, as well as the AWS service which can assist you in realizing your ideas. The first module, getting started, introduces ways to help expand your idea, find a co-founder, understand equity and compensation models, and debunk myths about getting started. The second module focuses on the customer. Specifically, how to find the right audience, understanding market needs, customer validation and building a prototype. The next topic covers the MVP and potential pitfalls. The pathway then moves to outlining the legal fundamentals and how to fundraise for your startup, including finding fundraising sources, creating a good pitch deck, and frequently asked questions about fundraising. Finally, growth strategies and company culture are outlined through understanding the stages of growth, building a product roadmap, common growing pains within a startup, and company culture.
